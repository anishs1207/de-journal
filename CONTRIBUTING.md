# Contributing to DeJournal

First off, thank you for considering contributing to DeJournal! It's people like you who make the decentralized web a better place.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please create an issue on GitHub. Include:
* A clear and descriptive title.
* Steps to reproduce the bug.
* What you expected to happen vs what actually happened.
* Your environment (browser, OS, wallet used).

### Suggesting Enhancements
Have an idea for a new feature? We'd love to hear it!
* Check if the feature has already been suggested.
* If not, open an issue and describe the enhancement in detail.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints.
6. Issue that pull request!

## Style Guide
* Use TypeScript for all logic.
* Use Tailwind CSS for styling.
* Follow the existing project structure.

## Code of Conduct
Please be respectful and professional in all interactions.

## Questions?
Feel free to open an issue or reach out to the maintainers.
